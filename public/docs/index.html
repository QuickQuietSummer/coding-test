Introduction

This documentation aims to provide all the information you need to work with our API.

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Base URL

http://backend.localhost

Authenticating requests

This API is not authenticated.

Get all requests

requires authentication

Only for employees.

Example request:
curl --request GET \
    --get "http://backend.localhost/api/requests?sort_date=quia&sort_status=repellat&start=at&end=et&filter_status=cupiditate&assigned_to_me=" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json"
const url = new URL(
    "http://backend.localhost/api/requests"
);

const params = {
    "sort_date": "quia",
    "sort_status": "repellat",
    "start": "at",
    "end": "et",
    "filter_status": "cupiditate",
    "assigned_to_me": "0",
};
Object.keys(params)
    .forEach(key => url.searchParams.append(key, params[key]));

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

fetch(url, {
    method: "GET",
    headers,
}).then(response => response.json());

GET api/requests

Query Parameters

sort_date  string optional  

Sorting by created. Values: new, old.

sort_status  string optional  

Sorting by status. Values: active, resolved.

start  string optional  

Start of date filter. Values: Carbon compatible string.

For example: 2010-1-1 00:00:00

end  string optional  

End of date filter. Values: Carbon compatible string.

For example: 2020-12-25 12:45:16

filter_status  string optional  

Filter by status. Values: active, resolved.

assigned_to_me  boolean optional  

Filter only requests assigned to your employee account.
